# Post drafts

Drafts of technical posts about engineering lessons from this
project, suitable for posting to dev.to, Hashnode, your own blog,
or as a HN Show post.

These are intended for **you to post from your own account**, not
for automated cross-posting. Coordinated cross-posting under one
identity reads as spam to platforms and triggers bans —
single-platform posts from a real account read as legitimate dev
sharing.

## Suggested platforms (pick one to start)

- **[dev.to](https://dev.to)** — friendly community for engineering
  writeups, easy to post, decent organic discovery if the title is
  good. Tags `#chrome`, `#extensions`, `#javascript`, `#webdev`.
- **[Hacker News Show](https://news.ycombinator.com/show)** — works
  if the post genuinely teaches something. The HN crowd will be
  honest about quality. Title has to deliver on what it promises;
  no clickbait.
- **r/programming** — works if the post is technically dense and
  not too self-promotional. Mods downvote anything that reads as
  "check out my project!" Rule of thumb: link to the post, not
  the project.
- **lobste.rs** (if you have an invite) — quality bar is higher
  than HN; if it fits there, it fits anywhere.

## What not to do

- Don't post the same article to multiple platforms in the same
  hour. Stagger by days; let each platform's audience react
  independently.
- Don't engage with downvotes. Engage with substantive criticism;
  ignore "I don't like this." Engaging with bad-faith comments
  burns your time and signal.
- Don't link the GitHub repo three times in the post. Once at the
  end (where I have it) is plenty.

## Format prep before posting

Most platforms want HTML or their own markdown dialect. Convert as
needed:

- **dev.to**: paste the markdown directly; it Just Works for
  headers, lists, code blocks, links.
- **HN**: HN doesn't render markdown. Post a link to the dev.to /
  blog version with a one-paragraph TL;DR in the comment.
- **Reddit**: paste the markdown; supports most of the syntax.
  Strip the front-matter HTML comment.
