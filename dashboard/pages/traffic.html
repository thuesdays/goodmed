How traffic is counted
Every Chrome instance we spawn installs a PerformanceObserver that emits one entry per completed resource load (HTML, CSS, JS, images, XHR, fetch, media, fonts — everything that goes over the network). Each entry carries transferSize — the exact byte count sent over the wire including TLS headers.

We aggregate by domain + hour bucket in memory, flushing totals to SQLite every 30 seconds. We never store the full URL — only the hostname — so this feature is safe on accounts where URL paths would leak sensitive IDs or query params.

Cached requests appear as 0-byte entries and still contribute to the request count, useful to spot cache-efficient vs cache-hostile sites.
