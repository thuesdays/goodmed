Parallel fires all N Chrome instances simultaneously — faster but needs N× RAM. Serial is gentler on resources.
Schedule
Base interval = remaining time in window / remaining runs. It gets a 50–150% base randomization, then an extra ±jitter% on top. With 30 runs between 07:00–20:00 the average gap is ~26 min; with 25% extra jitter each actual gap lands in ~10 min – 60 min.
Failure handling
After N consecutive runs exit with non-zero code, scheduler sleeps for the long pause duration. Between 1 and N-1 consecutive fails an exponential backoff (×2, ×4, ×8…) is applied to the interval.
