Scripts
Build runs as a sequence of actions — search queries, pauses, rotations, clicks, visits. Three scripts, each with its own role.
🎬 Main script 0
The entry point of a run. Sequenced loop-level actions: search a query, rotate proxy, pause, visit a warm-up URL, etc. When a search_query step runs and finds ads on the SERP, the Per-ad actions below execute once for each ad (before moving on to the next step).

Leave this script empty to use the legacy behavior: iterate every query in Domains → Search queries, apply per-ad actions to each ad, pause between queries.
🎯 Per-ad actions 0
Runs once per ad found by a search_query step in the Main script above. Steps execute in order for each ad — click, read, hover, go back, etc.

To treat your own or target domains differently, use the per-step flags on each action: skip my / skip target to exclude a step for those ads, or only my / only target to run it exclusively for those ads. Example: record every ad with a dwell step, but click only competitors — on your click_ad step check "skip if ad is on my domain" and "skip if ad is on target domain".
