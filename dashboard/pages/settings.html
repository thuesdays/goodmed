🧑 SERP engagement behavior
After ads are found on the SERP, Ghost Shell can simulate realistic user behavior (scroll, dwell, click an organic result) before moving on. This is the biggest lever for raising your ads/search hit rate — raw scraping triggers Google's anti-bot heuristics and cuts ad load on following queries. All settings here apply to every profile.
0.0 = never, 1.0 = after every query. 0.20-0.40 is realistic for commercial queries.
Organic dwell is how long we stay on the opened result. 8-25s matches p10-p75 of real click-through dwell times. Shorter looks like a bounce; longer is rare for quick scans.
⚡ Performance — resource blocking
Ghost Shell uses Chrome DevTools Protocol to block heavyweight URL patterns at the network layer. Ad detection works from the initial HTML, so images, fonts, video, and third-party trackers can be safely dropped — this cuts page-load time significantly on slow proxies.

Safety: none of these patterns touch google.com SERP HTML, core JS, or anything the ad parser needs. If you ever suspect blocking broke something, uncheck everything and reload.
One pattern per line. Uses CDP wildcard syntax: * matches any characters, prefix with scheme for host match (e.g. *://*.example.com/*). Invalid patterns are silently ignored by Chrome.

🔴 Auto-enrich from host Chrome — uses your personal browser data
⚠ DANGER ZONE — reads your personal browsing data.
When a fresh Ghost Shell profile starts for the first time, we automatically copy a random slice (30 days, ~500 URLs) of your real Chrome history + bookmarks into that profile. This makes the profile look like a real user to Google, but it also means your own browsing history — visited sites, bookmarks — ends up in the Ghost Shell profile folder.

Sensitive domains (banking, health, signed-in social) are always filtered out. Each Ghost Shell profile gets a different random slice of history, so they don't look identical to each other. Runs only once per profile — a sentinel file prevents re-imports on subsequent runs.
Disable this to start every new profile with a completely empty history. Profiles will look more "synthetic" to Google which may reduce ad load, but nothing personal leaves your browser.

📦 Export configuration
Download a single JSON file with every dashboard setting, all profile definitions, and both action pipelines. History is NOT included — runs, events, logs, captured competitor ads, IP history, and self-check results all stay behind. The result is a portable "setup snapshot" you can copy to another machine or check into git.
